\# Лабораторна робота №5. Наука про дані: обмін результатами (Streamlit Dashboard)



\---



\## Мета роботи

Головною метою роботи є розробка програмного забезпечення з веб-інтерфейсом (Dashboard) для візуалізації, динамічної фільтрації та зручного обміну результатами аналізу даних. Основний акцент зроблено на створенні інтерактивного середовища користувача (UI/UX) та оптимізації процесів обробки великих масивів даних за допомогою фреймворку Streamlit та бібліотек Matplotlib/Seaborn.



\## Архітектура Data Pipeline та оптимізація



Ядром додатку є модуль автоматизованого збору та обробки вегетаційних індексів (VCI, TCI, VHI) з серверів NOAA. Для забезпечення оптимізації та миттєвого відгуку інтерфейсу, архітектура завантаження реалізована за принципом кешування стану:

\- Процес мережевого парсингу 27 CSV-файлів, очищення даних (видалення NaN, реіндексація областей) та конкатенації виконується ієрархічно за допомогою декоратора `@st.cache\_data`.

\- Базовий DataFrame генерується в оперативній пам'яті лише один раз під час запуску локального сервера.

\- При взаємодії зі слайдерами та фільтрами, програма не виконує повторних I/O операцій з диском або мережею. Натомість вона математично маніпулює вже існуючим об'єктом у пам'яті. 

\- Такий архітектурний підхід гарантує виконання вимоги щодо продуктивності: зміна параметрів фільтрації викликає миттєве перемальовування графіків без хаотичних затримок на регенерацію датасету.



\## Проектування інтерфейсу (GUI) та логіка управління



Інтерфейс побудовано за класичним аналітичним патерном (Sidebar + Main View):

\- \*\*Панель керування (Sidebar):\*\* Забезпечує інтерактивний вибір часових рядів (Dropdown) та налаштування часових вікон за допомогою слайдерів (тижні та роки).

\- \*\*Обробка колізій сортування:\*\* Інтегровано чекбокси для сортування даних. Для запобігання логічним помилкам впроваджено захист: якщо користувач одночасно активує і зростання, і спадання, система генерує попередження (`st.warning`) та безпечно повертає дані до хронологічного відображення.

\- \*\*Управління станом (Session State):\*\* Кнопка "Очистити фільтри" використовує словник стану сесії для примусового скидання всіх інтерактивних віджетів до їх початкових (дефолтних) значень.



\## Візуалізація та аналітика



Для розділення логічних блоків використано систему вкладок (Tabs):

1\. \*\*Таблиця даних:\*\* Відображає відфільтрований сирий масив даних за допомогою оптимізованого віджета `st.dataframe`.

2\. \*\*Графік динаміки:\*\* Лінійний графік (Matplotlib), який відслідковує зміну обраного показника в часі. Для уникнення візуального шуму на осі абсцис реалізовано динамічний крок міток (відображення кожної 20-ї точки).

3\. \*\*Порівняння областей:\*\* Стовпчаста діаграма (Seaborn), що порівнює середні значення індексу. Для швидкої когнітивної ідентифікації цільова область підсвічується контрастним кольором (tomato) на тлі інших (steelblue), а також додано лінію середнього загальнонаціонального тренду.



\## Інструкція для запуску



1\. Схилюйте репозиторій та перейдіть у директорію `lab\_05`.

2\. Переконайтеся, що ви знаходитесь у віртуальному середовищі, та встановіть залежності: 

&#x20;  `pip install -r requirements.txt`

3\. Запустіть сервер веб-додатка командою: 

&#x20;  `python -m streamlit run lab5\_app.py`

4\. У браузері, що відкриється:

&#x20;  - Використовуйте бокову панель для налаштування цільового індексу та часових діапазонів.

&#x20;  - Перемикайтеся між вкладками для аналізу табличних даних, динаміки в часі та загальнонаціонального порівняння.

&#x20;  - Використовуйте кнопку очищення фільтрів для повернення системи до базового стану.
