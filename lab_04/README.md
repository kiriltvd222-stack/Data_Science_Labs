## Мета роботи
Головною метою роботи є розробка програмного забезпечення з графічним інтерфейсом (GUI) для візуалізації, моделювання та цифрової обробки сигналів. Основний акцент зроблено на розумінні природи адитивного шуму та практичному застосуванні методів цифрової фільтрації (Digital Signal Processing) за допомогою бібліотек Matplotlib та SciPy.

## Математична модель та архітектура генерації
Ядром програми є генератор гармонічного сигналу, що описується класичним рівнянням:
y(t) = A * sin(2 * pi * f * t + phase)

До ідеальної гармоніки застосовано адитивний гауссівський білий шум (AWGN). Для забезпечення оптимізації та коректного користувацького досвіду (UX), генерація шуму реалізована за принципом збереження стану:
- Базовий вектор нормалізованого шуму генерується в оперативній пам'яті лише один раз під час запуску програми. 
- При взаємодії зі слайдерами цей вектор математично масштабується — множиться на корінь із дисперсії (стандартне відхилення) та зсувається на величину математичного очікування (Mean). 
- Такий архітектурний підхід гарантує виконання вимоги: зміна параметрів самої гармоніки (амплітуди чи частоти) не викликає хаотичної регенерації шумового фону, дозволяючи об'єктивно оцінювати вплив фізичних параметрів на один і той самий сигнал.

## Аналіз методів цифрової фільтрації
Для відновлення зашумленого сигналу було інтегровано низькочастотний IIR-фільтр Баттерворта (scipy.signal.butter) 3-го порядку. 

Обґрунтування вибору технологій фільтрації:
1. Фільтр Баттерворта обрано через його максимально плоску амплітудно-частотну характеристику в смузі пропускання. Він не створює штучних пульсацій (ripple) на формі відфільтрованої гармоніки.
2. Проблема фазового зсуву: Будь-який стандартний IIR-фільтр зміщує сигнал у часі (фазове запізнення). Для подолання цього критичного недоліку застосовано функцію `scipy.signal.filtfilt`, яка виконує двосторонню фільтрацію (пряме та зворотне проходження). Це гарантує нульовий фазовий зсув — відфільтрована хвиля ідеально накладається на оригінальну "чисту" гармоніку без горизонтальних зміщень.
3. Частота зрізу (Cutoff Frequency) нормалізується відносно частоти Найквіста (половина від частоти дискретизації), що дозволяє гнучко відсікати високочастотні шумові компоненти.

## Проектування інтерфейсу (GUI) та оптимізація
Інтерфейс побудовано на базі інтерактивних віджетів Matplotlib:
- Головне вікно графіків: Забезпечує одночасний моніторинг трьох станів: ідеальний сигнал (синій), зашумлений сигнал (помаранчевий) та результат фільтрації (фіолетовий).
- Динамічне перемальовування: З метою оптимізації навантаження на CPU, при зміні параметрів на слайдерах програма не створює графік з нуля (функція plot), а лише оновлює масиви даних (метод set_ydata) у вже існуючих лініях та викликає draw_idle().
- Елементи стану: Кнопка "Reset" скидає всі 6 параметрів до стартових значень, а Checkbox дозволяє приховати зашумлений шар для чіткого візуального порівняння чистої та відфільтрованої гармонік.

## Інструкція для запуску
1. Схилюйте репозиторій та перейдіть у директорію lab_04.
2. Переконайтеся, що ви знаходитесь у віртуальному середовищі, та встановіть залежності:
   pip install -r requirements.txt
3. Запустіть основний скрипт:
   python lab4_harmonics.py
4. У вікні, що з'явиться:
   - Використовуйте блок слайдерів для налаштування хвилі та інтенсивності перешкод.
   - Змінюйте "Cutoff Freq", щоб керувати агресивністю фільтра: менші значення відсікають більше шуму, але при занадто малих значеннях амплітуда корисної хвилі також почне падати.
   - Зніміть прапорець "Show Noise", щоб приховати візуальний "бруд" і порівняти роботу фільтра Баттерворта з ідеальною моделлю.
